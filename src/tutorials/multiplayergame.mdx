# Writing a multiplayer game with shared world state.

A multiplayer game is typically characterized by taking place within a single world that all players can affect. Let's build one!

This is commonly implemented by setting up a coordinate system which represents locations within the world.  A simple key-value mapping stores the state of the world at a particular coordinate.

In this tutorial, we will write a very simple game with a shared world state. The world is represented as a square playing field and the only property that is available at each location is its 'color'. Some of you may recognize this as "place", which made its way around the Internet a while ago.

See and run a working solution here: https://studio.nearprotocol.com/?f=natqgjb1b

## Step 1 -- Start a new fiddle in NEARstudio

Go to https://studio.nearprotocol.com/ and start a new project (fiddle) by selecting "Token Smart Contract in AssemblyScript" and click "Create".

## Step 2 - Write a smart contract
In this simple game, we need only two actions: view the world state, and make changes to the state at particular coordinates.
In a more complex game with a large world, it may be important to not return the state of the whole world, but our game is small and simple enough that we don't have to worry about this.
Navigate to assembly/main.ts, and implement the following functions: setCoords and getCoords to operate on a single coordinate in the world. You will need to use globalStorage setItem and getItem functions.

```
export function setCoords(coords: string, value: string): void {
  globalStorage.setItem(coords, value);
}

export function getCoords(coords: string): string {
  return globalStorage.getItem(coords);
}
```
We also need a getMap function, which returns the full state of the game (we don't want to be making a separate call for every coordinate!)
```
export function getMap(): string {
  let num_rows = 10;
  let num_cols = 10;
  let total_cells = num_rows * num_cols;
  var arrResult:string[] = new Array(total_cells);
  let i = 0;
  for (let row=0; row<num_rows; row++) {
    //let cellEntry = "";
    for (let col=0; col<num_cols; col++) {
      let cellEntry = globalStorage.getItem(near.str(row) + "," + near.str(col));
      if (cellEntry == null) {
        cellEntry = "000000";
      }
      if (col < num_cols - 1) {
        cellEntry = cellEntry.concat(",".toString());
      }
      arrResult[i] = cellEntry;
      i++;
    }
    if (row < num_rows - 1) {
      arrResult[i-1] = arrResult[i-1].concat(";".toString());
    }
  }

  let result = arrResult.join("");
  return result as string;
}
```


## Step 2 -- Write a couple of tests for the contract
We can test the contract right away by writing some code in javascript. Open main.js and modify it to call the functions that we just wrote.

First let's call getMap. It's a function which does not modify the state, so we can call it through a callViewFunction interface.
```
async function getBoard() {
  const result = await near.callViewFunction(
    myAccountId, // we are using your default nearstudio account
    studioConfig.contractName, // contract name associated with this fiddle
    "getMap", // method name
    {}); // parameters - no parameters here
  return result;
}
```

Call this function from runTest() and try running it! You should be able to get the game state now!


Second, let's try to modify the game state! Add this to runTest() and try running it!
